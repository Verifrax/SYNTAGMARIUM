# SYNTAGMARIUM CONSTITUTION

SYNTAGMARIUM is the constitutional chamber of the VERIFRAX stack.

Its public function is to determine what may count, under which law version, for which admitted claim class, under which role boundary, under which invariant set, and under which transition discipline. It is the chamber that defines admissibility structure, not the chamber that accepts state, issues authority, executes runtime work, verifies receipts, recognizes terminal outcomes, or assigns recourse.

## Constitutional role

SYNTAGMARIUM owns the doctrine and object shape for:

- law versions
- claim classes
- role catalogues
- schema versions
- invariant sets
- transition rules
- projection contracts
- admission doctrine
- historical legal lineage for governed interpretation

A serious reader should be able to answer the question **"what may count?"** from SYNTAGMARIUM objects and doctrine without relying on founder narration.

## What SYNTAGMARIUM is not

SYNTAGMARIUM is not accepted state.  
SYNTAGMARIUM is not reconciliation runtime.  
SYNTAGMARIUM is not bounded subordinate cognition.  
SYNTAGMARIUM is not authority issuance.  
SYNTAGMARIUM is not execution.  
SYNTAGMARIUM is not verification.  
SYNTAGMARIUM is not recognition.  
SYNTAGMARIUM is not recourse.  
SYNTAGMARIUM is not intake or proof publication.

The chamber defines constitutional admissibility and legal structure only.

## Constitutional objects

The chamber is expected to publish or govern first-class objects across these surfaces:

### 1. Law versions
Law versions define the governing constitutional frame for admitted claim classes and downstream object interpretation.

Expected surface:
- `law/versions/`
- current law pointer
- historical law archive
- machine-readable law version objects

### 2. Claim classes
Claim classes define which bounded kinds of claims may be admitted into the governed path.

Expected surface:
- `claim-classes/`
- claim-class doctrine
- claim-class machine-readable objects
- admission constraints per class

### 3. Roles
Roles define which chamber or component may perform which bounded function and which functions are prohibited.

Expected surface:
- `roles/`
- role catalogue
- prohibited-role crossings
- sovereign vs subordinate separation

### 4. Invariants
Invariants define what must remain true across law, state, execution, verification, continuity, recognition, and recourse boundaries.

Expected surface:
- `invariants/`
- invariant definitions
- conflict rules
- preservation rules

### 5. Transitions
Transitions define how governed movement may occur between valid constitutional states without silently rewriting doctrine.

Expected surface:
- `transitions/`
- transition rules
- transition preconditions
- transition invalidation rules

### 6. Contracts
Contracts define projection and interpretation constraints for downstream chambers and external consumers.

Expected surface:
- `contracts/`
- projection contracts
- compatibility contracts
- downstream object-use contracts

### 7. Schemas
Schemas define the machine-readable admissibility and object structure required for reproducible external reading.

Expected surface:
- `schemas/`
- schema versions
- schema compatibility doctrine
- historical schema archive

## Admission doctrine

Admission into the VERIFRAX path is lawful only when a claim is admitted under:

1. an admitted claim class
2. a governing law version
3. a valid role boundary
4. the required invariant set
5. the allowed transition discipline
6. the declared projection contract

If one of these is absent, constitutional admissibility is incomplete.

## One active truth per governed subject

SYNTAGMARIUM doctrine requires singular current truth for governed constitutional objects. Historical material may exist, but historical material must never impersonate active law. Current law, current role doctrine, and current admission boundaries must be machine-distinguishable from historical snapshots.

## Historical continuity

Constitutional doctrine must remain reconstructable across time. That requires:
- historical law preservation
- schema lineage
- role lineage
- explicit distinction between active and historical constitutional material

A constitutional chamber is not durable if its governing frame cannot be reconstructed after platform loss or maintainer loss.

## Minimum completion standard

SYNTAGMARIUM is finish-grade only when an outsider can identify, from public objects and doctrine alone:

- which claim classes are admitted
- which law version governs
- which roles are permitted
- which invariants bind interpretation
- which transitions are lawful
- which contracts govern downstream use
- which material is current versus historical

Until that standard is met, constitutional rhetoric must not outrun constitutional object publication.

## Reading rule

Read SYNTAGMARIUM first when the question is:
- what may count
- under which law version
- under which role
- under which invariant
- under which transition
- under which contract

Do not read SYNTAGMARIUM as accepted state, runtime, authority issuance, execution, verification, recognition, or recourse of record.
